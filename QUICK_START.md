# ⚡ Быстрый старт: Запуск бота на сервере

## 🎯 Самый простой способ (5 минут)

### Вариант 1: Railway.app (рекомендуется)

1. Перейдите на https://railway.app
2. Войдите через GitHub
3. Нажмите "New Project" → "Deploy from GitHub repo"
4. Подключите ваш репозиторий (или создайте новый и загрузите файлы)
5. В настройках проекта добавьте переменную окружения:
   - **Name:** `BOT_TOKEN`
   - **Value:** `8137903259:AAG0VVcKfLDcOBdHQT_ADIR8s5daVu69eqU`
6. Railway автоматически запустит бота
7. Готово! Бот работает 24/7 🎉

### Вариант 2: Render.com

1. Перейдите на https://render.com
2. Войдите через GitHub
3. Создайте "Background Worker"
4. Подключите репозиторий
5. Настройки:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python bot.py`
6. Добавьте переменную окружения `BOT_TOKEN`
7. Готово!

## 📋 Необходимые файлы

Убедитесь, что в репозитории есть:
- ✅ `bot.py`
- ✅ `database.py`
- ✅ `requirements.txt`
- ✅ `Procfile` (для Heroku/Railway)

## 🔧 Переменные окружения

Обязательно установите:
- `BOT_TOKEN` - токен вашего бота

Опционально (если нужен прокси):
- `PROXY_URL`
- `PROXY_USERNAME`
- `PROXY_PASSWORD`

## ✅ Проверка работы

После развертывания:
1. Отправьте боту команду `/test` в Telegram
2. Если бот отвечает - всё работает!

## 📊 Мониторинг

- Логи можно посмотреть в панели управления платформы
- На Railway: вкладка "Deployments" → "View Logs"
- На Render: вкладка "Logs"

## 💾 Резервное копирование

База данных `expenses.db` будет сохраняться на сервере. Регулярно делайте бэкапы через панель управления платформы.

## 🆘 Проблемы?

1. Проверьте логи на платформе
2. Убедитесь, что токен правильный
3. Проверьте, что все файлы загружены
